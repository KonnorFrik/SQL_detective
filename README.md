# Detective Demo

Ты в роли детектива: по логам автоматизированной системы наблюдения за городом нужно вывести на чистую воду жителя, совершившего преступление. Запутанные улики, таблицы и связи — твой полигон для SQL-расследования.

## Быстрый старт

1. Поднять БД и применить миграции:
   ```bash
   cd container && docker compose up -d
   ```
2. Дождаться готовности Postgres (несколько секунд), после чего можно подключаться.

## Взаимодействие с системой

**Подключение к БД** (из корня проекта):
```bash
./cmd/connect/connect.sh
```
Либо вручную через `psql` с учётными данными пользователя `detective` (см. `container/.env`).

**Вьюшки** — основной способ изучать данные расследования. Список вьюшек в текущей схеме:
```text
\dv
```

**Проверка ответа** — функция с такой сигнатурой:
```sql
check_answer(incident_id_ INTEGER, guess_suspect_id INTEGER) RETURNS BOOLEAN
```
Подставь ID инцидента и ID подозреваемого; `true` означает верный ответ.
